# ADR-013: Logger Categories — @alt-javascript/camel-lite/* Idiom

**Status:** Accepted  
**Date:** 2026-03-27

## Context

Early in the project, logger categories used the `org.apache.camel.lite.*` namespace (Java reverse-domain convention). This is inappropriate for a JavaScript project: it implies affiliation with the Apache Software Foundation and Apache Camel, and creates confusion with log aggregation rules that already filter on `org.apache.camel.*` for upstream Camel instances.

## Decision

All logger categories follow the `@alt-javascript/camel-lite/{ClassName}` idiom, using sub-package path segments for nested components:

| Location | Category |
|----------|----------|
| Core components | `@alt-javascript/camel-lite/Pipeline` |
| master: component sub-classes | `@alt-javascript/camel-lite/master/FileLockStrategy` |
| Boot starter | `@alt-javascript/camel-lite/boot/CamelLiteAutoConfiguration` |
| CLI runtime | `@alt-javascript/camel-lite/cli/CamelRuntime` |

The npm package scope `@alt-javascript` unambiguously identifies the project. The path after the slash mirrors the package and class structure.

## Consequences

**Positive:**
- Logger categories unambiguously identify the camel-lite project and cannot be confused with Apache Camel.
- The `@alt-javascript/camel-lite/*` prefix matches the npm package namespace convention familiar to JavaScript developers.

**Negative:**
- Breaking change for any existing log aggregation dashboards, alerts, or filters that were configured for `org.apache.camel.lite.*` categories. Those rules must be updated.
