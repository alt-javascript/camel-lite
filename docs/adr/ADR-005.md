# ADR-005: HTTP Component — Producer Only

**Status:** Accepted *(not revisable)*  
**Date:** 2026-03-26

## Context

Camel's `http4` and `netty-http` components implement both a producer (outbound HTTP calls) and a consumer (embedded HTTP server). In Node.js, HTTP server frameworks — Express, Fastify, Hono, Koa — are mature, widely used, and highly optimised. Wrapping one of them inside a Camel consumer would add framework selection opinion, lifecycle complexity, and request-mapping boilerplate without providing meaningful value over using the framework directly.

## Decision

`camel-lite-component-http` implements only the producer (outbound `fetch`-based HTTP calls). There is no HTTP server consumer. Inbound HTTP is handled at the framework level of the caller's choice. For CDI-managed inbound HTTP wired into camel-lite routes, the `boot-camel-lite-hono` integration provides a Hono-based adapter.

## Consequences

**Positive:**
- Simple, focused component with a clear scope.
- The producer uses native `fetch` — zero runtime dependencies.
- Framework agnostic for inbound traffic: the caller chooses Express, Fastify, Hono, or anything else.

**Negative:**
- No inbound HTTP consumer in camel-lite itself. Users who want request-driven Camel routes must wire their framework handler to a `direct:` or `seda:` endpoint manually.
