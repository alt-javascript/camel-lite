# ADR-010: Test Framework — node:test Built-in

**Status:** Accepted  
**Date:** 2026-03-26

## Context

Jest, Mocha, and Vitest are popular Node.js test frameworks. Each adds external dependencies and introduces module system complexity. Jest in particular requires CJS transforms for ESM code (or experimental VM modules), which conflicts with the pure-ESM mandate (ADR-002). Node 18+ ships a built-in test runner (`node:test`) with TAP-compatible output and `assert` from `node:assert/strict`.

## Decision

All test suites use the built-in `node:test` runner with `assert` from `node:assert/strict`. No external test framework is installed in any component package.

```js
import { test } from 'node:test';
import assert from 'node:assert/strict';

test('exchange carries the body', () => {
  const ex = new Exchange({ body: 'hello' });
  assert.equal(ex.in.body, 'hello');
});
```

## Consequences

**Positive:**
- Zero test dependencies — `package.json` `devDependencies` stays empty for test tooling.
- Works natively with ESM — no transform pipeline, no flags.
- Fast startup and execution.

**Negative:**
- Less ecosystem tooling: no jest-dom matchers, no snapshot testing, less IDE integration (no test lens in most editors out of the box).
- TAP output format requires a TAP formatter (or `--reporter=spec`) for human-readable output; some CI systems may need a TAP-to-JUnit converter.
