# ADR-009: SQLite — node:sqlite Built-in Over better-sqlite3

**Status:** Accepted  
**Date:** 2026-03-26

## Context

The `sql:` component needs a synchronous SQLite API (Camel's SQL component is synchronous at the exchange level). The most widely used Node.js SQLite library is `better-sqlite3`, which provides exactly this API. However, `better-sqlite3` requires native C++ compilation. On macOS with Apple Clang and Node 24, this fails due to C++20 requirements in the V8 headers. Node 22.5.0 introduced a built-in `node:sqlite` module with a synchronous `DatabaseSync`/`StatementSync` API that is API-compatible with `better-sqlite3`.

## Decision

`camel-lite-component-sql` uses `node:sqlite` (`DatabaseSync`, `StatementSync`). There is no dependency on `better-sqlite3` or any other native SQLite binding.

One API difference to note: `StatementSync.all()` takes spread arguments (`stmt.all(...values)`) rather than an array (`stmt.all(values)`). The producer unpacks arrays with spread before calling `StatementSync` methods.

## Consequences

**Positive:**
- Zero native compilation dependencies — installs reliably on any platform where Node 22.5+ is available.
- No `node-gyp` or compiler version matrix to worry about.

**Negative:**
- `node:sqlite` carries an `ExperimentalWarning` at import time — the API may change in a future Node LTS.
- Requires Node 22.5+ (Node 22 LTS). Not available on Node 18 or Node 20.
